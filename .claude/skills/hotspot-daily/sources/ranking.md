# Ranking Rubric

排序规则：Agent 读取此文件，对每条候选热点打分。

## 评分维度（5 维，每维 0-5 分）

### 1. 时效性 (Timeliness) — 权重 20%

| 分数 | 标准 |
|------|------|
| 5 | 今天发生/发布，尚未被广泛报道 |
| 4 | 今天发生，已有一定报道 |
| 3 | 24 小时内 |
| 2 | 48 小时内 |
| 1 | 本周内 |
| 0 | 超过一周 |

### 2. 影响面 (Impact) — 权重 25%

| 分数 | 标准 |
|------|------|
| 5 | 影响整个 AI 行业格局（新模型发布、重大政策） |
| 4 | 影响多个细分领域（开源模型、平台变更） |
| 3 | 影响特定领域的从业者 |
| 2 | 影响小众群体 |
| 1 | 仅学术兴趣 |
| 0 | 无明显影响 |

### 3. 可行动性 (Actionability) — 权重 25%

| 分数 | 标准 |
|------|------|
| 5 | 读者可以立即采取具体行动（迁移、试用、调整策略） |
| 4 | 本周内应该做出响应 |
| 3 | 值得关注并纳入规划 |
| 2 | 了解即可，无需行动 |
| 1 | 纯知识性信息 |
| 0 | 无可行动性 |

### 4. 可信度 (Credibility) — 权重 15%

| 分数 | 标准 |
|------|------|
| 5 | 官方发布 + 多家权威媒体确认 |
| 4 | 权威媒体首发 + 社区验证 |
| 3 | 可靠来源，但尚未被广泛验证 |
| 2 | 单一来源，需谨慎 |
| 1 | 传言/泄露，未经证实 |
| 0 | 可疑来源 |

### 5. 受众匹配 (Audience Match) — 权重 15%

目标受众：AI 开发者、技术创业者、产品经理、技术决策者。

| 分数 | 标准 |
|------|------|
| 5 | 直接影响目标受众的日常工作 |
| 4 | 影响目标受众的中期规划 |
| 3 | 目标受众应该知道 |
| 2 | 与目标受众间接相关 |
| 1 | 仅边缘相关 |
| 0 | 不相关 |

## 加权总分计算

```
total = timeliness * 0.20
      + impact * 0.25
      + actionability * 0.25
      + credibility * 0.15
      + audienceMatch * 0.15
```

## 选择规则

1. 按 total 降序排列
2. 选 total 最高的为 **primary**（主选）
3. 选第 2、3 名为 **alternates**（备选），用于 QC Gate 失败时替补
4. 如果 top1 和 top2 分数差 <0.3，优先选可行动性更高的
5. 同一来源（source）不应同时占据 primary 和 alternate

## 输出要求

对每条候选输出：
- 5 个维度的分数
- 加权总分
- `rankReason`：一句话说明为什么选/不选它
