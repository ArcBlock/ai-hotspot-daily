# Style Guide

写作规范：Agent 生成 page.json 内容时遵循此指南。

## 标题规则

- **title（主标题）**：简短有力，中文 ≤20 字，英文 ≤10 词
  - 聚焦事件主体 + 一个核心关键词
  - 示例："GLM-5 开源：745B 全昇腾模型登顶开源第一"
- **subtitle（副标题）**：补充结论/行动导向，中文 ≤30 字，英文 ≤15 词
  - 告诉读者"这对你意味着什么"
  - 示例："开发者需要重新评估开源模型选型"
- 主标题用于社交分享时单独成立
- 页面上两行显示：主标题大字 + 副标题小字
- **必须是结论式标题**：告诉读者"这意味着什么"，不是"发生了什么"
- 错误示例："OpenAI 发布了 GPT-5"
- 正确示例："GPT-5 发布：开发者现在应该重新评估你的 LLM 选型策略"

## 语言风格

- 简洁、直接、工程师友好
- 避免营销语气（不用"震惊""颠覆""革命性"）
- 避免模糊表述（不用"可能会""据说""或许"）
- 用具体数据支撑（"性能提升 40%"优于"性能大幅提升"）
- 比较 API 价格时，keyMetrics 中必须分开列出 input 和 output 两行价格（不要合并），并在 note 中提示潜在差异（如 tokenizer 不同、输出质量差异可能导致实际 token 消耗不同），避免"A 是 B 的 1/N"这种简单倍数对比误导读者
- 中文内容中的英文术语保持原文（如 LLM、API、fine-tuning）

## 内容结构要求

### keyMetrics（关键数据对比，可选）
- 当事件涉及可量化对比时提供（如模型性能、API 价格、benchmark 分数）
- 2-5 行数据，每行包含：指标名、2-4 个对比对象的值
- 高亮标注"本次事件主角"列
- 只放硬数据，不放主观评价
- 每行可选 `note` 字段：当指标存在口径差异或需要补充说明时使用（如 tokenizer 不同导致 token 消耗不同）
- **价格对比强制规则**：当对比 API 价格时，必须同时列出 input 和 output 两行（不要合并为单行）。如果某个模型的 tokenizer 效率差异显著，必须在 note 中说明
- 示例场景：新模型发布时对比竞品的分数/价格/速度

### whatHappened（发生了什么）
- 3-5 行 Markdown
- 回答：什么事？谁做的？什么时候？
- 只陈述事实，不加评论

### whyItMatters（为什么重要）
- 回答：跟读者有什么关系？
- 必须连接到读者的实际工作场景
- 不要泛泛而谈

### whoIsAffected（受影响人群）
- 列出 2-4 个具体群体
- 每个群体标注影响程度（high/medium/low）
- 说明具体影响是什么
- 如果事件与个人开发者/独立开发者相关，必须包含一个面向他们的条目，说明对"日常使用 LLM API 写代码、构建个人项目"等场景的具体影响，不要只关注企业和专业团队

### whatToDo（行动清单）
- **immediate**（立即）：今天就能做的，≥1 条
- **thisWeek**（本周）：本周应该做的，≥1 条
- **watchAndWait**（观望）：持续关注但不急于行动的，≥1 条
- 每条行动必须具体可执行，不是"关注动态"这种空话

### options（策略对比，可选）
- 当有多个应对方案时提供
- 每个方案说明：适合谁、成本、风险
- **必须包含一个"保持现状"选项**：不做任何改变的方案，说明这样做的成本（零）和风险（可能错过什么）
- 保持现状选项放在最后
- 不做推荐，让读者自己判断

### risks（风险与误区）
- ≥2 条
- 指出容易犯的错误或常见误解
- 包含"不要做什么"的建议

### sources（来源）
- ≥3 个独立来源，其中至少 1 个为一手来源（官方博客、Model Card、论文、官方公告）
- 每个来源附摘录要点（不是全文复制）
- 标注发布者和发布时间
- 每个来源必须标注 `tier` 字段：`"official"`（一手/官方发布）或 `"media"`（媒体转述），帮助读者判断信息可靠程度

## 双语规则

- `zh` 和 `en` 内容独立撰写，不是逐字翻译
- 中文面向中文读者的语境和关注点
- 英文面向英文读者的语境和关注点
- 两个版本的核心结论和行动清单应一致

## 置信度 (confidence) 计算

```
base = 30
+ 来源数量加分: min(sourceCount * 8, 40)
+ 来源权威度加分: hasOfficialSource ? 15 : 0
+ 多源一致性加分: sourcesAgree ? 15 : 0
+ 时间衰减: eventAge < 48h ? -15 : 0
confidence = min(base + 加分 + 时间衰减, 100)
```

> 设计意图：事件发布不足 48 小时时，社区尚未大规模验证，实际上限约 85%。只有经过充分验证的事件才可能接近 100%。

## 禁止事项

- 不复制原文大段内容（版权约束）
- 不在标题中使用问号（结论式，不是提问式）
- 不使用 emoji
- 不使用第一人称（"我们认为"）
- 不做投资建议
