# Ranking Rubric

排序规则：Agent 读取此文件，对每条候选热点打分，选出 1 个主打话题（hero）+ 5-6 个卡片话题（cards）。

## 评分维度（5 维，每维 0-5 分）

### 1. 全民热度 (Popularity) — 权重 30%

多平台出现的话题优先。

| 分数 | 标准 |
|------|------|
| 5 | 4+ 个平台同时出现，全网刷屏 |
| 4 | 3 个平台出现，热搜/热榜上有名 |
| 3 | 2 个平台出现，有一定讨论度 |
| 2 | 1 个平台出现，但热度较高 |
| 1 | 1 个平台出现，热度一般 |
| 0 | 只在 1 个平台出现，热度低 |

### 2. 可视化潜力 (Visualization Potential) — 权重 25%

有硬数据可做图表的话题优先。

| 分数 | 标准 |
|------|------|
| 5 | 有具体数字/排名/趋势，天然适合图表化 |
| 4 | 有可量化的对比或时间序列数据 |
| 3 | 有部分数据，可以构造可视化 |
| 2 | 数据较少，但可以用简单图形呈现 |
| 1 | 几乎无数据，只能用文字描述 |
| 0 | 纯观点/情绪，完全无数据支撑 |

### 3. 话题新鲜度 (Freshness) — 权重 20%

今天刚爆发的优先。

| 分数 | 标准 |
|------|------|
| 5 | 过去 6 小时内爆发的新话题 |
| 4 | 今天（12 小时内）爆发 |
| 3 | 24 小时内出现 |
| 2 | 48 小时内出现 |
| 1 | 本周内持续发酵的话题 |
| 0 | 持续多天的老话题，无新进展 |

### 4. 情绪极性 (Polarity) — 权重 15%

引发讨论和立场分化的话题优先。

| 分数 | 标准 |
|------|------|
| 5 | 有明确"赞/反"两方观点，引发激烈讨论 |
| 4 | 有明显争议点，评论区两极分化 |
| 3 | 有一定讨论度，但立场较为一致 |
| 2 | 引发关注但讨论不深入 |
| 1 | 轻度关注，无明显情绪 |
| 0 | 无明确立场的中性信息 |

### 5. 领域多样性 (Diversity) — 权重 10%

主打和卡片不重叠领域。

| 分数 | 标准 |
|------|------|
| 5 | 补充缺失领域，增加选题多样性 |
| 4 | 所在领域仅有 1 个候选 |
| 3 | 所在领域有 2 个候选 |
| 2 | 所在领域有 3 个候选 |
| 1 | 所在领域已有多个候选被选中 |
| 0 | 已有多个同领域候选被选中 |

## 加权总分计算

```
total = popularity * 0.30
      + vizPotential * 0.25
      + freshness * 0.20
      + polarity * 0.15
      + diversity * 0.10
```

## 领域分类 (Category)

为每条候选分配一个领域分类，可选值：

- **科技** — 科技产品、AI、互联网、数码
- **娱乐** — 影视、综艺、明星、游戏
- **体育** — 赛事、运动员、体育政策
- **社会** — 民生、法律、教育、公共事件
- **财经** — 股市、经济政策、商业动态
- **生活** — 美食、旅行、健康、消费
- **国际** — 国际关系、海外新闻、全球事件
- **文化** — 历史、艺术、学术、文学

## 选择规则

1. 按 total 降序排列
2. **Top 1 → hero**（主打信息图）：选加权总分最高的话题
3. **Top 2-7 → cards**（热点卡片）：从剩余候选中按分数依次选取 5-6 个
4. cards 中需要 **>= 3 个不同 category**
5. 如果 cards 中同一 category 超过 2 个，用下一名候选替换
6. hero 和 cards 的 category **应尽量不重复**（hero 的 category 在 cards 中最多出现 1 次）

## 图表类型建议

Ranker 应为每个选中的话题建议适合的图表类型。

### Hero 图表类型 (suggestedChartType)

根据话题数据特征选择最合适的图表：

| 数据特征 | 图表类型 |
|----------|----------|
| 有排名/榜单 | `bar` |
| 有时间序列/趋势 | `line` |
| 有占比/构成 | `ring` |
| 有事件先后顺序 | `timeline` |
| 有 A vs B 对比 | `versus` |
| 有地域分布 | `heatmap` |

### Card 迷你图表类型 (suggestedMiniChartType)

卡片使用简化的迷你图表：

| 数据特征 | 迷你图表类型 |
|----------|-------------|
| 有排名/榜单 | `miniBar` |
| 有趋势变化 | `trend` |
| 有进度/百分比 | `progress` |
| 核心是一个大数字 | `bigNumber` |
| 有 A vs B 对比 | `versus` |
| 有标签/关键词集合 | `tags` |

## 输出要求

对每条候选输出：
- 5 个维度的分数（popularity / vizPotential / freshness / polarity / diversity）
- 加权总分（total）
- 领域分类（category）
- 图表类型建议（hero 用 suggestedChartType，card 用 suggestedMiniChartType）
- `rankReason`：一句话说明为什么选/不选它
